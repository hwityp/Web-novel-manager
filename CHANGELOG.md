## [v1.3.17] - 2026-02-21

### Added
- **소스 폴더 즉시 적용 기능 (GUI):**
  - `[소스 폴더에 저장]` 버튼을 새로 추가하여, 2단계 파싱 완료 후 장르 추론 단계 없이 원본 소스 파일의 이름을 즉시 정규화된 이름으로 교체(`os.rename`)할 수 있는 기능을 제공합니다.
  - `일괄 처리` 모드 진행 시에도 이 즉시 적용 단계를 매끄럽게 거쳐가도록 파이프라인 흐름을 보강했습니다.

### Fixed
- **한글 자소 분리 및 시각적 변형(야민정음) 복원:**
  - NFD 분리뿐 아니라 필터 우회 목적으로 작성된 변형(`ㄷH공ㅂlㄱr`, `ㄹㅔㅂ 1` 등) 텍스트를 파이프라인 내에서 자동으로 원래 한글(`대공비가`, `렙 1`)로 조립해주는 커스텀 로직 `compose_korean_jamo`를 추가했습니다.
  - 영문자 기반 파일에서는 오작동하지 않게 예외 조건을 강화했습니다.
- **파일명 내 단일 숫자 추출 오류 해결:**
  - `렙 1 이지만`과 같이 단위가 없고 문자가 바로 뒤따르는 숫자를 불필요하게 '범위(연재 화수)'로 오인하여 추출하는 정규식 버그를 해결했습니다.

## [v1.3.16] - 2026-02-03

### Fixed
- **구글 검색 장르 추론 개선:**
  - **심층 스크래핑 로직 강화:** 검색 스니펫에서 장르 정보가 부족하거나 너무 일반적인 경우(예: '소설'), **정보성 사이트(디시인사이드, 나무위키, 아카라이브 등)를 자동으로 스크래핑**하여 본문 내용을 분석하도록 개선했습니다.
  - **장르 우선순위 적용:** 본문에서 여러 장르가 발견될 경우, 더 구체적이고 특징적인 장르를 우선 선택합니다.
    - 예: `현대인 선협세계 빙의물` -> '무협/판타지' 대신 **[선협]** 채택
    - 예: `울트라맨 팬픽...` -> '판타지' 대신 **[패러디]** 채택

## [v1.3.10] - 2026-02-03

### Changed
- **복합 파일명 정규화 지원 강화:**
  - `1부 완결 2부` 등이 섞여 있는 복잡한 파일명 패턴(`1-546 完 2부 212`)을 감지하여, 순서를 억지로 바꾸지 않고 원본 흐름을 보존(`1-546 (완) 2부 1-212`)하도록 개선했습니다.
  - 개선된 로직은 복합 패턴 이후에 나오는 `외전`, `후기` 등의 부가 정보도 놓치지 않고 정확히 파싱합니다.

## [v1.3.9] - 2026-02-03

### Changed
- **폴더 정리 기능 개선:**
  - 폴더 평탄화(Flatten) 로직을 강화하여 서브폴더와 압축 파일 내부의 파일들을 모두 최상위(Root) 폴더로 바로 꺼내도록 수정했습니다. `[26.02]...` 같은 불필요한 서브폴더가 생성되지 않습니다.
  - 폴더 정리 후 파일 목록을 다시 스캔할 때, 백업 폴더(`Temp`)가 포함되지 않도록 제외 처리하여 리스트가 정확해졌습니다.
  - "1. 폴더 정리" 버튼이 Dry-Run(미리보기) 전용에서 실제 실행(Action) 모드로 변경되었습니다.
- **파일명 정규화 로직 개선:**
  - 숫자나 단위 뒤에 공백 없이 붙은 완결 마커(`1부完`, `5권완`)가 정상적으로 인식되도록 정규식 패턴을 수정했습니다. (결과: `1부 (완)`)
- **일괄 처리(Batch) 개선:**
  - "일괄 처리" 버튼 실행 시에도 수정된 폴더 평탄화 및 재스캔 로직이 정확히 적용되도록 프로세스를 통합했습니다.

### Added
- **CSV 저장 기능:** 기존 "CSV 열기" 버튼을 "CSV 저장"으로 변경하여, 현재 작업 목록을 CSV 파일로 내보낼 수 있습니다.
- **폴더 열기 기능 복구:** "폴더 열기" 버튼이 스마트 폴백(Smart Fallback)을 통해 가장 적절한 폴더(Target > Source > Default)를 열도록 수정했습니다.

## [1.3.8] - 2026-02-02

### ⚡ 개선 사항
- **폴더 정리 기능 (Folder Organizer)**:
  - **평탄화 및 백업 (Flatten & Backup)**: '폴더 정리' 실행 시 하위 폴더의 파일들을 최상위 폴더(Root)로 이동시키고, 원본 폴더 구조는 `Temp` 폴더로 안전하게 백업합니다.
  - **연계성 강화**: 폴더 정리 후 변경된 파일 위치를 자동으로 인식하여 다음 단계(장르 분석, 정규화)와의 파이프라인 연계를 보장합니다.
- **정규화 로직 (Normalization Logic)**:
  - **복합 완결 패턴 개선**: `完+ 외전` 같은 복합 패턴을 `(완) + 외전`으로 정확히 분리합니다.
  - **숫자+단위 파싱 개선**: `1920 2부`와 같이 숫자 뒤에 단위(부, 권, 화)가 붙을 때 숫자가 제목에 남는 오류를 수정했습니다.
  - **추가 외전/노이즈 태그**: `외포`(외전 포함) 키워드 인식 및 `(AI 번역)` 노이즈 제거 패턴을 보강했습니다.

## [1.3.5] - 2026-01-02

### ⚡ 개선 사항
- **정규화 로직 (Normalization Logic)**:
  - **(19N) 태그 보존**: 파일명에서 `(19N)` 태그가 더 이상 제거되지 않습니다.
  - **범위 정규화**: `001-242` 스타일의 범위를 `1-242`로 정규화합니다 (선행 0 제거).
  - **미분류 장르 처리**: 장르를 알 수 없는 경우 `[미분류]` 태그를 생략합니다 (예: `[미분류] Title.txt` 대신 `Title.txt`).
